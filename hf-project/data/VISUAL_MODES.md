# 视觉模式使用规则

## 概述
视频工厂支持6种视觉模式，每种模式适用于不同内容类型。选择正确的模式能让画面更匹配内容，提升观看体验。

## 模式详解

### 1. hero（英雄模式）
**适用场景**：
- 视频开场（第一句话）
- 视频结尾（CTA/总结）
- 重要结论/核心观点

**视觉特征**：
- 大标题居中
- 背景渐变+光效
- 简洁有力

**示例**：
```
口播："今天给大家分享一个赚钱的新思路"
视觉模式：hero
```

### 2. dashboard（仪表盘模式）
**适用场景**：
- 数据指标展示
- 排行榜/排名
- 多指标对比（3个以上指标）

**视觉特征**：
- 卡片式布局
- 数据可视化（数字、进度条、趋势箭头）
- 信息密度高

**示例**：
```
口播："销售额增长20%，用户数突破10万，复购率达到35%"
视觉模式：dashboard
```

**什么时候用**：
- 有3个以上数据指标
- 需要展示趋势/排名
- 信息量大，需要结构化展示

### 3. compare（对比模式）
**适用场景**：
- 有明确两方对立数据
- A vs B 的对比
- 数量/比例对比

**视觉特征**：
- 左右分栏
- 对比色（红vs绿，涨vs跌）
- 突出差异

**示例**：
```
口播："30人里20个选择A方案，只有10个选B方案"
视觉模式：compare
```

**触发条件（严格）**：
- 必须有两方明确的数据
- 必须有数量/比例对比
- "但是"、"然而"不算对比

### 4. flow（流程模式）
**适用场景**：
- 步骤/流程展示
- 时间线/发展历程
- 操作指南

**视觉特征**：
- 箭头连接
- 步骤编号
- 进度指示

**示例**：
```
口播："第一步注册账号，第二步完善资料，第三步开始赚钱"
视觉模式：flow
```

**什么时候用**：
- 有明确的步骤顺序
- 需要展示流程
- 操作指南类内容

### 5. quote（引用模式）
**适用场景**：
- 引用名人名言
- 金句/观点
- 强调某句话

**视觉特征**：
- 引号装饰
- 大字排版
- 简洁背景

**示例**：
```
口播："正如马云所说，今天很残酷，明天更残酷"
视觉模式：quote
```

### 6. hud（HUD模式）
**适用场景**：
- 未来感/科技感内容
- 预测/趋势分析
- 数据监控界面

**视觉特征**：
- 半透明边框
- 科技感线条
- 动态扫描效果

**示例**：
```
口播："2025年AI行业趋势预测"
视觉模式：hud
```

## 选择规则速查表

| 内容类型 | 推荐模式 | 关键词 |
|----------|----------|--------|
| 开场/结尾 | hero | "今天"、"最后"、"总结" |
| 数据指标 | dashboard | "增长"、"达到"、"突破" |
| A vs B | compare | "vs"、"对比"、"里...个" |
| 步骤指南 | flow | "第一步"、"然后"、"最后" |
| 名人名言 | quote | "正如...所说"、"名言" |
| 未来预测 | hud | "预测"、"趋势"、"未来" |

## 错误示例

### ❌ 错误：乱用compare
```
口播："他做了三件事：第一件事，第二件事，第三件事"
错误模式：compare
正确模式：flow（这是流程，不是对比）
```

### ❌ 错误：该用dashboard却用hero
```
口播："销售额2000万，用户10万，增长率50%"
错误模式：hero
正确模式：dashboard（多指标，需要结构化展示）
```

### ❌ 错误：该用compare却用dashboard
```
口播："30人里20个选A，10个选B"
错误模式：dashboard
正确模式：compare（两方对比，需要突出差异）
```

## 实现建议

1. **自动选择**：LLM根据口播内容自动选择模式
2. **人工校验**：关键场景（开场、结尾）人工确认
3. **灵活调整**：根据实际效果调整规则
